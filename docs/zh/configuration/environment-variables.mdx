所有配置均通过 `.env` 进行。复制 `example.env` 并填入相应的值：

```bash
cp example.env .env
```

<div id="configuration-levels">
  ## 配置级别
</div>

每个集成都有一个配置级别，用于表明其重要程度：

| 级别 | 含义 | 未配置时的行为 |
|-------|---------|------------------------------|
| **必填** | 核心系统依赖 | 系统会报错——聊天和主要功能将无法使用 |
| **推荐** | 重要功能支持项 | 优雅降级——该功能会明确显示为不可用，但系统仍可运行 |
| **可选** | 增强能力 | 无感降级——系统可正常运行，只是该能力不存在 |

> **注意**：管理员配置的模型（Admin → Models 页面）可替代 LLM 环境变量。健康检查会同时考虑这两种来源。

<div id="frontend-local-dev-only">
  ## 前端（仅限本地开发）
</div>

前端有一个单独的 env 文件，**仅用于本地开发**：`frontend/.env.local`。

> **Docker 中不会使用此文件。** 在 Docker 容器内，Next.js 会在内部将 `/api/*` 代理到 Python 后端（端口 8000 仅在容器内部可用），因此前端不需要 env 文件。

对于本地开发，默认配置开箱即用——你**不**需要创建 `frontend/.env.local`，除非你的后端运行在非默认端口上。

如果需要覆盖默认配置，请手动创建 `frontend/.env.local`：

```bash
echo 'NEXT_PUBLIC_API_URL=http://localhost:9000' > frontend/.env.local
```

| 变量 | 默认值 | 说明 |
|---|---|---|
| `NEXT_PUBLIC_API_URL` | `http://localhost:8000` *(auto)* | **浏览器**用于直接调用 API 的后端 URL（OAuth 重定向、流式传输）。如果未设置，则会从 `window.location` 自动检测——仅当你的后端在本地运行于非标准端口时，才需要覆盖此值。 |

> **构建时说明**：`NEXT_PUBLIC_*` 变量会在执行 `pnpm build` 时固化到 JS bundle 中。在运行时修改它们（例如通过根目录的 `.env`）不会生效——这就是为什么它们仅用于本地开发，并放在 `frontend/.env.local` 中。

### 扩展思考（Reasoning）

设置 `LLM_REASONING_EFFORT` 后，FIM One 会启用模型的扩展思考能力，并将内部思维链展示在 UI 的"思考中"步骤中。FIM One 使用 [LiteLLM](https://github.com/BerriAI/litellm) 自动将推理强度参数转换为各提供商的原生格式。

#### 支持的提供商

| 提供商 | `LLM_BASE_URL` | 工作方式 | 是否返回推理内容？ |
|---|---|---|---|
| **OpenAI**（o1 / o3 / o4-mini） | `https://api.openai.com/v1` | 原生发送 `reasoning_effort` | 是 |
| **Anthropic**（Claude 3.7+） | `https://api.anthropic.com/v1/` | LiteLLM 通过原生 Anthropic API 路由，发送 `thinking` 参数 | 是 |
| **Google Gemini**（2.5+） | `https://generativelanguage.googleapis.com/v1beta/openai/` | 通过兼容端点发送 `reasoning_effort` | 是 |

LiteLLM 根据 `LLM_BASE_URL` 自动检测提供商，并映射到正确的 API 格式。未知 URL 按 OpenAI 兼容方式处理。

#### 重要注意事项

<Warning>
**第三方代理 / 自定义端点不保证兼容。**
如果你的 `LLM_BASE_URL` 指向第三方 API 代理（如 OpenRouter、one-api、自定义网关），LiteLLM 会尝试根据 URL 进行正确路由。但如果你的代理期望非标准格式，推理功能可能无法正常工作。请查阅代理的文档了解其期望的参数格式。
</Warning>

#### 推理模式下的温度限制

部分提供商在启用推理时会对温度参数有特殊限制：

- **Anthropic**：启用扩展思考后，要求 `temperature=1`。如果使用 Anthropic 配合扩展思考，你**必须**设置 `LLM_TEMPERATURE=1`——Anthropic 会拒绝其他温度值。
- **OpenAI GPT-5.x**：始终仅支持 `temperature=1`。LiteLLM 的 `drop_params` 参数过滤会自动处理——不支持的温度值会被静默丢弃。GPT-5.x 无需用户手动操作。

#### `LLM_REASONING_BUDGET_TOKENS` 的工作方式

此变量**主要对 Anthropic 路径有效**。设置后，它会覆盖自动计算的预算，通过 LiteLLM 作为 `budget_tokens` 发送到 `thinking` 参数中。未设置时，预算根据 `LLM_MAX_OUTPUT_TOKENS` x effort 比例自动计算：

| `LLM_REASONING_EFFORT` | 预算比例 | 示例（max_tokens = 64000） |
|---|---|---|
| `low` | 20% | 12,800 tokens |
| `medium` | 50% | 32,000 tokens |
| `high` | 80% | 51,200 tokens |

最低预算为 1,024 tokens（Anthropic 的硬性下限）。

对于 OpenAI 和 Gemini，提供商根据 `reasoning_effort` 级别内部处理 token 分配——`LLM_REASONING_BUDGET_TOKENS` 无效。

<div id="agent-execution">
  ## 智能体执行
</div>

| 变量 | 必填 | 默认值 | 说明 |
|---|---|---|---|
| `REACT_MAX_ITERATIONS` | 否 | `20` | 每个 ReAct 请求的最大工具调用迭代次数 |
| `MAX_CONCURRENCY` | 否 | `5` | DAG 执行器中的最大并行步骤数 |
| `DAG_STEP_MAX_ITERATIONS` | 否 | `15` | 每个 DAG 步骤内的最大工具调用迭代次数 |
| `DAG_MAX_REPLAN_ROUNDS` | 否 | `3` | 目标未达成时的最大自主重新规划尝试次数 |
| `DAG_REPLAN_STOP_CONFIDENCE` | 否 | `0.8` | 当智能体判断目标无法实现的置信度超过此阈值时，将停止重试（`0.0` = 永不提前停止，`1.0` = 任意失败时立即停止） |
| `DAG_TOOL_CACHE` | 否 | `true` | 在单次 DAG 执行内缓存相同的工具调用。按 key 的异步锁防止惊群效应。设为 `false` 禁用缓存 |
| `DAG_STEP_VERIFICATION` | 否 | `false` | 在每个 DAG 步骤完成后启用基于 LLM 的质量检查。验证失败时，步骤会携带验证反馈重试一次。增加延迟但提高结果准确性 |
| `AUTO_ROUTING` | 否 | `true` | 启用自动查询分类。Fast LLM 分析每个查询，将其路由到 ReAct（单步骤）或 DAG（多步骤）执行模式。`/api/auto` 端点使用此功能。设为 `false` 则需手动选择执行模式 |

| `REDIS_URL` | 否 | — | 跨进程中断中继的 Redis 连接 URL。Docker Compose 自动配置，仅本地多进程开发时需手动设置（`WORKERS>1`）。 |
| `WORKERS` | 否 | `1` | Uvicorn 工作进程数。`1` 为安全默认值，无需额外服务。`>1` **必须使用 PostgreSQL**（SQLite 为单写者）。认证、OAuth、文件操作均为多进程安全（基于 JWT）。**限制**：未配置 `REDIS_URL` 时，流式中断/注入仅在同一工作进程内有效。Docker Compose 自动配置 Redis。 |

<div id="oauth-optional">
  ## OAuth（可选）
</div>

当某个提供商同时配置了 `CLIENT_ID` 和 `CLIENT_SECRET` 时，登录页面会自动显示对应的 OAuth 按钮。

| 变量 | 必填 | 默认值 | 说明 |
|---|---|---|---|
| `GITHUB_CLIENT_ID` | 否 | — | GitHub OAuth App 的客户端 ID。可在 [github.com/settings/developers](https://github.com/settings/developers) → OAuth Apps 创建 |
| `GITHUB_CLIENT_SECRET` | 否 | — | GitHub OAuth App 的客户端密钥 |
| `GOOGLE_CLIENT_ID` | 否 | — | Google OAuth 客户端 ID。可在 [console.cloud.google.com/apis/credentials](https://console.cloud.google.com/apis/credentials) 创建 |
| `GOOGLE_CLIENT_SECRET` | 否 | — | Google OAuth 客户端密钥 |
| `DISCORD_CLIENT_ID` | 否 | — | Discord OAuth2 客户端 ID。可在 [discord.com/developers](https://discord.com/developers/applications) 创建 |
| `DISCORD_CLIENT_SECRET` | 否 | — | Discord OAuth2 客户端密钥 |
| `FEISHU_APP_ID` | 否 | — | Feishu (Lark) App ID。可在 [open.feishu.cn](https://open.feishu.cn/app) 创建。需要 `contact:user.email:readonly` 权限 |
| `FEISHU_APP_SECRET` | 否 | — | Feishu (Lark) App Secret |
| `FRONTEND_URL` | **生产环境** | `http://localhost:3000` | OAuth 完成后浏览器最终跳转到的地址。生产环境中必须设置（例如 `https://yourdomain.com`） |
| `API_BASE_URL` | **生产环境** | `http://localhost:8000` | 外部可访问的后端 URL，用于构建 OAuth 回调 URL。生产环境中必须设置 |
| `NEXT_PUBLIC_API_URL` | **生产环境** | *(自动检测为 `<hostname>:8000`)* | 浏览器端用于 OAuth 重定向的 API 基础 URL。**这是一个前端构建时变量**——本地开发时请在 `frontend/.env.local` 中设置；如果是自定义生产部署，则作为 Docker 构建参数传入。自动检测适用于标准反向代理配置（80/443 端口）。 |

> **生产环境** = 本地可选（默认值即可），但对于**任何面向互联网的部署都必须设置**。

<div id="oauth-callback-urls-to-register-with-each-provider">
  ### 需要向各提供商注册的 OAuth 回调 URL

| 提供商     | 需注册的回调 URL                                               |
| ------- | -------------------------------------------------------- |

| Google  | `https://yourdomain.com/api/auth/oauth/google/callback`  |
| Discord | `https://yourdomain.com/api/auth/oauth/discord/callback` |

<div id="analytics-optional">
  ## 分析统计（可选）
</div>

所有分析提供商均为可选项。可任意组合设置——所有已启用的提供商会同时加载。全部留空即可完全禁用分析功能（本地开发建议禁用）。

| 变量 | 必填 | 默认值 | 说明 |
|---|---|---|---|
| `NEXT_PUBLIC_GA_MEASUREMENT_ID` | 否 | — | Google Analytics 4 衡量 ID（例如 `G-XXXXXXXXXX`）。可在 [analytics.google.com](https://analytics.google.com) 获取 |
| `NEXT_PUBLIC_UMAMI_SCRIPT_URL` | 否 | — | Umami 分析脚本 URL（例如 `https://your-umami.com/script.js`）。自托管、注重隐私的替代方案——[umami.is](https://umami.is) |
| `NEXT_PUBLIC_UMAMI_WEBSITE_ID` | 否 | — | Umami 网站 ID。设置了 `NEXT_PUBLIC_UMAMI_SCRIPT_URL` 时必填 |
| `NEXT_PUBLIC_PLAUSIBLE_DOMAIN` | 否 | — | Plausible 分析域名（例如 `yourdomain.com`）。轻量级、注重隐私——[plausible.io](https://plausible.io) |
| `NEXT_PUBLIC_PLAUSIBLE_SCRIPT_URL` | 否 | `https://plausible.io/js/script.js` | 自托管 Plausible 实例的自定义脚本 URL |

> 所有 `NEXT_PUBLIC_*` 分析变量均为**构建时变量**——修改后需要重新构建前端才能生效。
